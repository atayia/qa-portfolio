REQ-STR-01: PaymentIntent Creation — Happy Path

To Do
Endpoint
POST /v1/payment_intents
Description
The API shall create a PaymentIntent and return HTTP 200 when provided a valid amount, currency, and payment method. The response must include id, status: "succeeded", amount, and currency.

REQ-STR-02: Card Decline Error Contract

To Do
Endpoint
POST /v1/payment_intents
Description
The API shall return HTTP 402 with a structured error object when a card is declined. The error object must include error.type, error.code: "card_declined", and error.decline_code. Generic and specific decline reasons (e.g. insufficient_funds) must be surfaced distinctly.

TC-PI-002, TC-PI-003

REQ-STR-03: Charge Creation & Retrieval

To Do
Endpoint
POST /v1/charges  •  GET /v1/charges/:id
Description
The API shall create a charge and return HTTP 200 with status: "succeeded" and paid: true. A created charge must be retrievable by its ID with all fields intact, including object: "charge" and correct amount.

REQ-STR-04: Refund Processing

To Do
Endpoint
POST /v1/refunds
Description
The API shall process full and partial refunds against a succeeded charge. Full refunds must set refunded: true on the charge object. Partial refunds must correctly reflect amount_refunded without setting refunded: true until the full amount is returned.

To Do

REQ-STR-06: Input Validation & Error Responses

To Do
Endpoint
POST /v1/payment_intents  •  POST /v1/charges
Description
The API shall return HTTP 400 with error.type: "invalid_request_error" and a meaningful error.param field for invalid inputs, including zero or negative amounts, unsupported currency codes, and missing required fields.
